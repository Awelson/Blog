    in order to construct a term of type ‚àèn:‚Ñï, B(n). Thus we infer the following elimination rule :

    \[ \frac{n:‚Ñï \vdash \text{B}(n):\text{Type}, \quad b_0:\text{B}(0), \quad b_s:\prod_{n:\mathbb{N}}\text{B}(n)\to \text{B}(s(n))}{rec_\mathbb{N}(b_0,b_s):\prod_{n:\mathbb{N}}\text{B}(n)} \]

    Alternatively we could have removed b‚ÇÄ:B(0) and b‚Çõ:‚àèn:‚Ñï, B(n) ‚Üí B(s(n)) from the top and replaced the bottom with

    rec_‚Ñï : B(0) ‚Üí [‚àèn:‚Ñï, B(n) ‚Üí B(s(n))] ‚Üí [‚àèn:‚Ñï, B(n)]

    both versions are equivalent. I just find the top one a bit cleaner.

    When B(n) does not actually depend on the choice of n:‚Ñï then we get the simplified version :

    \[ \frac{\text{B}:\text{Type}, \quad b_0:\text{B}, \quad b_s:\mathbb{N}\to \text{B}\to \text{B}}{rec_\mathbb{N}(b_0,b_s):\mathbb{N}\to \text{B}} \]

    Addition function¶

    We do the same thing all over again but with B := ‚Ñï ‚Üí ‚Ñï this time. This will be a lot more difficult, but the principle is the same. First, notice that

    Wait a moment! what the hell is + 0 ??? How does this make sense? I acknowledge that this is all a bit unconventional but I promise it makes sense, give me a chance to explain.

    Let's ask the simple question first : What is the type of + 0? Well, given that +:‚Ñï‚Üí‚Ñï‚Üí‚Ñï and 0:‚Ñï, we must have + 0:‚Ñï‚Üí‚Ñï. This is simply the elimination rule for the function type. Moreover, one of the computation rules state that :

    where +(n,m) := n + m, this is simply writing + with the Polish notation which does nothing but to emphasize that + is indeed a function. This is helpful because most education systems present and teach + as an operator, and the infix notation n + m makes it harder to make the connection that + can be thought of as a function.

    It is not that + being an operator is untrue, but rather that "operator ‚äÜ function" and we wish to look at things from a broader perspective.

    Going back on topic, this means that

    + 0 := Œªm:‚Ñï, +(0,m) := Œªm:‚Ñï, m

    Again, the first input is useless, it actually only becomes useful when we apply the original (actually dependent) version of the elimination rule for ‚Ñï.
